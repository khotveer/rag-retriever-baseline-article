# RAG Retriever Baseline

This repository contains the implementation of a **baseline retriever for Retrieval-Augmented Generation (RAG) systems**.

The code demonstrates how to build a simple retrieval pipeline using:

- Document chunking
- Embedding generation
- FAISS vector search
- Retrieval evaluation using **Precision@K, Recall@K, and MRR**

This implementation accompanies the article:

**"The Most Important Component of RAG: Building and Evaluating a Retriever"**

---

# How to Execute

### Step 1 — Configure API Key

Edit the `.env` file and add your Gemini API key.



---

### Step 2 — Run the Notebook

Run the notebook **end-to-end**.



The notebook performs the following steps:

1. Load the input document  
2. Perform section-based chunking  
3. Generate embeddings  
4. Store embeddings using FAISS  
5. Perform similarity search  
6. Evaluate the retriever using standard retrieval metrics

# Note

This repository is created as a **demo implementation for the accompanying article**, so the evaluation results are generated using a **small sample dataset**.

In real-world production systems:

- Evaluation is performed on **larger labeled datasets**
- Additional improvements are typically applied such as:
  - Hybrid search (**BM25 + vector search**)
  - Reranking models
  - Query expansion techniques
